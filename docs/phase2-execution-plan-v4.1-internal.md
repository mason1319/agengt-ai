# Aggie速记英语（星伴英语）Phase 2 执行计划（v4.1 内部版）

## 1. 目标与边界

### 1.1 阶段目标
在不回滚第一阶段闭环的前提下，推动“文档可执行化 + 运营可持续交付”的最小增量交付。

### 1.2 不回退原则
- 保持第一阶段已验收闭环不变：登录、公开咨询、学生任务、老师干预、家长摘要、创始人对账。
- 不新增未经确认的 SaaS 商业化能力、不引入 App/小程序正式分发。
- 不新增无来源控件，不允许“仅前端占位按钮”。
- 仅在现有 React + Vite + Cloudflare Pages/Functions + D1 + R2 框架下推进。

### 1.3 参照源
- PRD 主源：`docs/phase1-prd-starmate-english.md`
- 内部业务补充：`docs/StarMate-PRD-v4.1-internal-CN.md`
- 交付标准：`docs/international-delivery-standards-v4.1-internal.md`
- 执行约束：`docs/phase1-execution-index-v4.1-internal.md`

## 2. Phase 2 本周里程碑（最小闭环）

### 第 1 周：稳定性与可复现性补齐
1. 统一前后端接口执行基线
   - 固化 `stack:verify` 的运行约束与可复现参数（`verify:smoke` 仅作为后端已启动时快速复测）。
   - 所有本地/预发验证优先使用 `npm run stack:verify`。
2. 缺口复核
   - 再次同步 `docs/ui-control-api-field-map.csv` 与 `docs/phase1-chain-gap-checklist-v4.1-internal-CN.md` 之间的差异。
   - 标记“尚未有后端接口支持但页面仍保留按钮/操作”的项目。
3. 产出验收证据
   - 每条缺口形成 `P1 待修` 列表，附对应接口路径与测试样例。

### 第 2 周：运营支撑闭环
1. 公开咨询与试听页闭环检查
   - 咨询列表、领取建议、试听预约、负责人接管三链路完整记录。
2. 创始人运营对账体验统一
   - 学员收入、课时、点名三张明细的关联导出字段口径保持一致。
   - 统一错误提示和空态提示文案。
3. 风险告警和AI用量
   - 确保 AI 用量、到期风险、异常学生告警显示有真实来源。

### 第 3 周：体验优化（P2 之前）
1. 课程/练习/个人中心页面二次对齐（仅视觉微调）
   - 不引入新模块，不扩大业务范围。
2. 术语与品牌口径统一
   - “星伴英语/ Aggie速记英语”命名与课程/家长入口文案统一。

## 3. P0 / P1 / P2 修复优先级

### P0（72h 内必须清零）
- 任何已登录用户越权查看他人学生、家长、课时、收费数据。
- `/api/v1/auth/login` 与 `/api/v1/me` 链路失效。
- 与 D1/R2 关键链路断连导致核心页面空白或异常退出。
- 公开端可直接读取敏感内部字段。

### P1（1 周内修复，不阻断已上线功能）
- 页面出现无接口绑定控件（按钮点击无请求 / 404）。
- 关键闭环接口字段口径不一致导致页面渲染出错。
- 任务、点名、课时和咨询状态转移无审计链。
- 角色域数据源不一致（demo 与生产 API 差异未解释）。

### P2（阶段内可记录，不阻塞当前验收）
- 表格样式统一、动画、主题细化、暗色微调。
- 文化墙二级审核流程、AI 模型参数可视化面板。
- 高级导出报表（PDF/Word/多维透视）与图表美化。

## 4. 交付前置条件（每次收口必检）

- `npm run validate:contracts` 通过。
- `npm run stack:verify` 通过。
- `npm run lint` / `npm run typecheck` / `npm run build` 通过（如变更代码）。
- 三元一致：`docs/openapi-v4.1.yaml`、`schema.sql`、`docs/ui-control-api-field-map.csv` 不出现高风险不一致。
- 所有新增/修改接口具备 smoke 检查用例，优先写入 `docs/phase2-api-smoke-checklist.md`。

## 5. 风险与应对（不扩张）

- 线上可见性优先：所有验证都以真实页面动作+接口日志为证据，不接受静态截图替代。
- 避免“UI先行”：先接口闭环后再做视觉补充。
- 避免代理/本地端口差异导致误判：固定 `stack:verify` 为验收命令。
