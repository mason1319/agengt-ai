# 星伴英语 StarMate v4.1 文档审计结论

**日期**：2026-06-06
**审计对象**：PRD、设计文档、研发 SOP、运营 SOP、合规索引
**结论**：已可作为第一阶段开发依据。第一阶段必须先交付本机构 Web/H5/PWA 可用系统，不进入 App、小程序、公开 SaaS 和在线支付。

---

## 1. 本次发现并已修正的问题

| 问题 | 风险 | 处理结果 |
|---|---|---|
| PRD 仍写“第一阶段只证明四件事” | 会漏掉招生、课程、课时经营闭环 | 改为七个闭环：学习、老师、家长、创始人、招生、课程、课时 |
| 文档适用范围只写内部用户 | AI 客服和公开课程表没有正式位置 | 增加外部咨询客户，但仅限咨询、课程查看、试听预约 |
| 运营 SOP 写“暂不公开推广” | 与课程展示、招生咨询冲突 | 改为不做在线销售闭环、平台招商、在线支付、大规模投放 |
| AI 客服边界写“不承诺价格” | 与公开课程费用展示冲突 | 改为只能展示公开课程价格，不承诺提分、录取、保留名额、特殊优惠 |
| Figma 范围缺公开课程和试听 | 设计阶段可能漏页面 | 增加公开咨询、公开课程表、课程详情、试听预约 |
| 家长缴费和课时未贯穿验收 | 家长最关心的信息可能看不到 | 增加缴费记录、剩余课时、已消课、保留课时验收 |
| 老师点名消课未贯穿验收 | 课时账目仍可能混乱 | 增加点名、请假、缺勤、迟到、消课、保留课时闭环 |
| 创始人对账未贯穿验收 | 收费、课时、报名无法管理 | 增加课程、学生、家长维度的收费课时对账验收 |
| 设计验收仍偏页面完整 | 容易做成静态界面 | 增加控件、接口、字段、状态、经营闭环验收 |
| 合规索引未覆盖咨询客户 | 外部咨询采集联系方式有合规风险 | 增加外部咨询客户适用范围和咨询合规底线 |

## 2. 当前产品边界

| 允许做 | 不允许做 |
|---|---|
| 本机构账号登录 | 公开自助注册 |
| 学生 AI 学习任务 | 社交、排行榜、公开社区 |
| 老师授权学生管理 | 老师查看全机构学生和完整家长电话 |
| 家长查看自己孩子 | 家长查看其他孩子数据 |
| 创始人驾驶舱 | 多机构平台代理 |
| AI 客服咨询 | 自动承诺提分或录取 |
| 公开课程表和公开价格 | 在线支付和自动开课 |
| 试听预约 | 自动保留名额 |
| 线下收费记录 | 支付套餐售卖 |
| 课时账户和点名消课 | 无审计的课时修改 |

---

## 3. 第一阶段必须交付的七个闭环

| 闭环 | 最小可用结果 |
|---|---|
| 学生学习闭环 | 学生登录后看到今日任务，完成口语/复习，结果落库 |
| 老师干预闭环 | 老师只看授权学生，能处理异常并提交干预 |
| 家长反馈闭环 | 家长只看自己孩子，能看成长摘要、周报、课时余额 |
| 创始人管理闭环 | 创始人能看活跃、异常、老师处理、AI 用量和经营风险 |
| 招生咨询闭环 | 客户能通过 AI 客服咨询，提交线索，预约试听 |
| 课程收费闭环 | 课程表展示时间、年级、班型、公开价格、名额，后台记录收费 |
| 课时打卡闭环 | 老师点名后系统记录到课、请假、缺勤、迟到，并消课或保留 |

## 5. 下一步工作建议

第一步不应该继续改 UI，也不应该直接写页面。应先做：

1. `openapi-v4.1.yaml`
2. `schema.sql`
3. `ui-control-api-field-map.csv`

原因：用户明确要求每个按钮、弹窗、输入框、统计卡片都必须绑定真实接口和 D1 字段。没有这三份文件，继续写前端会重新变成空按钮和假页面。
