## 已清理
- 课程/缴费：`订单号待更新` -> `订单号待核对`
- 课程与缴费：`课次时间未确认` -> `课次时间待更新`
- 机构：`机构名称待核对` -> `机构名称未设置`
- 阶段总结：`阶段总结待生成` -> `阶段总结生成中`
- 师资模块：`师资资料待补充...` -> `师资资料待完善...`
- 家校反馈兜底文案：
  - `待补充` -> `教师待完善` / `反馈待提交`
  - `家长与学员评价待补充。` -> `家校反馈待提交。`
- 空态文案：`课程数据暂未同步` -> `课程数据未同步`

## 当前保留（业务语义）
以下仍为状态语义词，不建议移除，建议后续统一映射到后端枚举：
- `待确认`（任务/流程状态）
- `待完成`（反馈、练习完成态）
- `待更新`（状态更新提示）
- `年级未录入`、`班型未设置`、`价格未设置`、`课程时间未设置`

## 建议（下一轮）
1. 将所有状态口径集中到后端返回枚举（`pending/processing/completed/...`）并做一次“状态词映射表”统一渲染。
2. 将 `待确认 / 待完成 / 待更新` 改为更窄层次的业务定义（例如：`待处理`、`进行中`、`待审核`）后统一替换。
