# StarMate Phase 1 Kick-off 检查清单（v4.1 内部版）

本文用于启动第一个 Sprint 前确认团队共识。Kick-off 不是口头会议纪要，必须形成可追踪的维护人、验收标准和未决问题清单。

## 1. 会议目标

- 确认 Phase 1 当前只做内部机构自用 Web/H5/PWA 版本。
- 确认主 PRD、v4.1 内部补充、执行标准、架构、设计、SOP、合规文件的分工。
- 确认 Sprint 1 只围绕登录、角色闭环、控件映射、真实 API/D1 数据和公开咨询边界推进。
- 确认 v4.0 PRD 仅作为历史归档，不作为当前开发验收源。

## 4. Sprint 1 验收标准

| 链路 | 验收标准 | 证据 |
| --- | --- | --- |
| 登录 | `POST /api/v1/auth/login` 成功后 token 持久化，`GET /api/v1/me` 可识别角色 | 成功/失败请求示例，浏览器截图 |
| 学生 | 今日任务可拉取，练习可提交，结果可回填 | API 请求证据，D1/R2 记录，学生端截图 |
| 老师 | 能查看授权学生异常并提交干预 | 权限拒绝示例，干预写入记录，老师端截图 |
| 家长 | 只能查看绑定孩子课程、课时、成长摘要 | 越权失败示例，家长端截图 |
| 创始人 | 能查看看板、线索、课时/收费对账 | API 请求证据，导出/审计记录 |
| 公开咨询 | 可提交咨询、AI/规则化回应、试听预约，不出现在线支付/公开注册 | 公开端截图，线索写入记录 |

## 5. 必跑门禁

```bash
npm run validate:contracts
npm run stack:verify
```

若命令无法运行，必须记录：命令、失败原因、影响范围、替代证据、后续负责人。
