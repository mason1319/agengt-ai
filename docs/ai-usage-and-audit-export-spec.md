# AI 用量与审计导出字段说明（来源告警）

## 适用范围
- 文件名：`starmate-ai-usage-YYYY-MM-DD.csv`
- 文件名：`starmate-ai-audit-YYYY-MM-DD.csv`
- 生成入口：平台端「平台 AI 用量监控」页面
  - 导出机构用量 -> `starmate-ai-usage-*.csv`
  - 导出机构审计 -> `starmate-ai-audit-*.csv`

## 1) 导出区块结构
每个 CSV 由两段组成：
1. 明细表（首段）
2. `[来源告警汇总]` 汇总区（尾段）

### 1.1 明细表头
- 机构用量导出：
  - `机构名称, 套餐, 套餐编码, 窗口内AI消耗, 月度配额, 请求数, 最后请求时间, AI来源, 来源分类`
- 审计导出：
  - `时间, 机构, 机构ID, 角色, 用户ID, 动作, 决策, 延迟(ms), Tokens, 理由, 来源, 来源分类, IP, 请求参数, 响应参数`

## 2) 来源告警汇总说明
- 每次导出均追加以下固定字段（CSV 内文本行）：
  - `整体来源风险`
  - `指标`
  - `值`
  - `阈值`
  - `状态`

### 2.1 字段定义
- `整体来源风险`
  - 规则：`模拟来源请求占比 >= 模拟阈值` 或 `来源未知请求占比 >= 未知阈值` 时为 `告警`。
- `真实来源请求占比`
  - 值：`真实来源请求 / 总来源请求 * 100%`
  - 阈值：`-`
  - 状态：`-`
- `模拟来源请求占比`
  - 值：`模拟来源请求 / 总来源请求 * 100%`
  - 阈值：`当前模拟阈值`，默认 `35%`
  - 状态：`告警` / `正常`
- `来源未知请求占比`
  - 值：`来源未知请求 / 总来源请求 * 100%`
  - 阈值：`当前未知阈值`，默认 `15%`
  - 状态：`告警` / `正常`
- `来源总请求`
  - 来源分类汇总后的总请求数
- `真实来源请求`
- `模拟来源请求`
- `来源未知请求`

## 3) 统计口径
- 机构用量导出 `来源` 摘要使用 AI 用量页的 `sourceSummaryForExport`
  - 已按后端返回的来源统计（`real`、`mock`、`unknown`）预聚合。
- 审计导出 `来源` 汇总使用当前导出页行级记录计数（每条审计记录记为一次请求）。

## 4) 适用人群
- 平台管理员：用于季度稽核、费用分摊、异常来源监控。
