# 上篇 — 工程宪法

## 0.1 技术栈

| 层 | 技术 | 选型理由 |
|-------|-----------|-----------|
| **学生端** | React + Vite + TypeScript + shadcn/ui（PWA） | 响应式移动优先；可安装 PWA，通过 Service Worker 支持离线 |
| **老师/管理后台** | React + Vite + TypeScript + shadcn/ui | 与学生端同一代码库；桌面优先，响应式 |
| **家长端** | 微信小程序原生（WXML/WXSS） | 微信生态分发必需 |
| **静态托管** | Cloudflare Pages | Git 关联自动部署；全球 CDN；push 即发布 |
| **API / 后端** | Cloudflare Pages Functions（`/api/v1`） | Workers 原生；全球边缘节点；无冷启动 |
| **数据库** | Cloudflare D1（SQLite 兼容） | 无服务器关系型数据库；零运维 |
| **文件存储** | Cloudflare R2 | 录音文件、生成内容、导出报表、文化墙素材 |
| **缓存** | Cloudflare KV | 会话、热数据、限流 |
| **AI（第一阶段）** | Mock + 规则引擎 | 供应商抽象层；第一阶段不依赖真实大模型 |
| **AI（第三阶段）** | 通义千问 / Kimi / 讯飞（主）+ OpenAI / Claude（兜底） | 国内优先，全球兜底 |
| **CI/CD** | GitHub Actions → Cloudflare Pages | `lint → typecheck → build → wrangler dry-run → deploy` |
| **可观测性** | 全部响应携带 `trace_id` + `request_id` | 结构化日志；审计表 |

> **第一阶段约束**：AI 为模拟/规则化。所有供应商接口统一抽象在适配层，后续接入真实大模型时，适配层之外零代码改动。

## 0.2 UI 设计与交付标准

### 0.2.1 设计流水线（必须遵守）

| 步骤 | 工具 / 流程 | 规则 |
|------|---------------|------|
| 1. 结构交互层 | Figma——页面流程、组件库 | UI 冻结后方可写代码 |
| 2. 设计交付 | Figma 导出：布局图、交互说明、状态图、组件清单 | — |
| 3. 组件映射 | 设计组件 → **shadcn/ui** 官方组件 | shadcn/ui 没有等价组件时才可自定义 |
| 4. 样式落地 | shadcn/ui 主题 Token + Tailwind CSS 变量 | 项目中 **禁止存在独立的 `.css` 文件** |
| 5. 集成验证 | 页面控件 → API 字段映射回归 | 每个交互控件必须追溯到 API |

### 0.2.2 设计硬约束

| # | 约束 | 执行方式 |
|----|-----------|-------------|
| 1 | Figma 稿未冻结 + 控件-API 映射表未完成 → 禁止进入开发 | 合并阻断 |
| 2 | 不允许自定义 CSS 体系替代 shadcn/ui。样式扩展仅允许通过主题 Token 或 Tailwind 原子类 | Code Review |
| 3 | 所有页面在 375px 宽度（iPhone SE）不得出现横向滚动条 | QA 门禁 |
| 4 | 每个交互元素（按钮、输入框、弹窗触发）必须绑定真实 `/api/v1` 接口。无后端动作的控件不允许合并 | API 审查 |
| 5 | 禁止"边写代码边改图"。必须先完成完整控件标注，再按映射表开发 | 流程门禁 |
| 6 | 智能体相关按钮必须标注：触发对象（学生/老师/家长）、失败降级路径、重试机制 | 设计审查 |

### 0.2.3 每轮设计交付物

| 交付物 | 格式 | 说明 |
|-------------|--------|-------------|
| `ui-wireframes/` | Figma / PNG | 页面级线框图 |
| `ui-components/` | Figma / Markdown | 组件规范 |
| `ui-flow.md` | Markdown | 按钮事件与导航逻辑 |
| `ui-control-map.csv` | CSV / XLSX | 控件→API→字段映射 |
| `ui-states/` | Figma / Markdown | 每个页面的空状态、加载态、成功态、失败态（含重试入口） |

### 0.2.4 UI 质量要求

- **状态覆盖**：每个关键控件必须渲染：空状态、加载状态、成功状态、错误状态（含重试按钮）。
- **移动优先**：断点至少覆盖 375px 和 768px。
- **信息层级**：不允许出现"大面积空白但堆满控件"的无效页面。每个页面要有明确的主操作。
- **无障碍**：Phase 2 目标：`eslint-plugin-jsx-a11y` 零告警。

## 0.3 开发流程

### 0.3.1 开发顺序（强制）

1. **接口合约先行**，再做页面开发。
2. 合同级验证：请求/响应 Schema、鉴权、错误码、幂等、审计字段。
3. 前端对接已验证的接口合约。
4. 移动端 / 小程序接入同一套 API。

### 0.3.2 Mock 管控与淘汰

| 阶段 | 允许范围 |
|-------|-----------------|
| P0（设计评审） | UI 视觉样稿 + 示例数据 |
| P1（业务页面） | 除 AI 接口外，不得使用硬编码 mock 数据 |
| P2（AI 接口） | Mock 数据必须标注来源，必须可回退 |
| 生产环境 | **生产链路中禁止出现任何 mock 数据** |

> 展示 mock 数据的控件在非生产环境中必须打上 `[Mock]` 标记。

## 0.4 代码质量与冗余控制

### 0.4.1 硬规则

- 禁止未引用的函数、组件、样式、字段映射、路由、参数、配置。
- 同一职责 → 复用统一服务/组件。
- 开发分支禁止保留"临时兜底脚本"和"未归档测试数据"。
- 禁止长期保留废弃页面或组件。

### 0.4.2 扫描流水线

| 扫描类型 | 工具 / 命令 | 频率 |
|-----------|---------------|-----------|
| 静态扫描 | ESLint + TypeScript strict + `no-unused-vars` / `no-unused-imports` | 每次提交前 |
| 类型检查 | `tsc --noEmit` | 每次提交前 |
| 无用代码 | `knip` + `ts-prune` | 每轮迭代 |
| 依赖 | `npm audit` / `depcheck` | 每轮迭代 |
| 安全 | `npm audit` + 基础漏洞规则 | 每轮迭代 |
| 包体积 | `vite-bundle-visualizer` | PR 审查时 |

### 0.4.3 质量门禁（阻断级）

| 门禁 | 标准 |
|------|----------|
| Lint | 零告警 |
| 类型检查 | 零错误 |
| 构建 | 通过（`npm run build`） |
| 无用代码 | 每轮迭代自动发现 + 人工确认删除 |
| 关键告警 | 零：未使用变量、未使用导入、重复声明、死代码 |
| 关键接口测试 | 2xx / 4xx / 5xx 分支覆盖 |
| 扫描报告 | 每次合并前上传 |

### 0.4.4 问题处理

- 24 小时内提交修复计划。
- 无法立即修复 → 记录技术债并设截止日。不得以"功能里程碑完成"口径通过。

## 0.5 多端兼容

| 平台 | 覆盖范围 | 优先级 |
|----------|----------|----------|
| 桌面 Web（PC） | 全部主流程 | P0 |
| 移动 Web（H5） | 375px 响应式；无横向溢出 | P0 |
| PWA（学生端） | 可安装；离线缓存近期内容 | P0 |
| 微信小程序 | 核心学习闭环：任务、口语、复习 | P1 |
| 原生 iOS / Android | 推迟到 Phase 3；PWA 覆盖 Phase 1–2 | P2 |

> **跨端最低一致性**：术语、任务状态、错误文案、反馈语言在所有端上必须一致。

## 0.6 数据与安全合规

- 所有环境必须记录：来源、授权、审计主键。
- 所有跨端接口通过统一鉴权策略。
- API 错误禁止静默吞掉。必须返回标准错误格式并提示恢复动作。
- 所有查询必须带 `institution_id`（从 JWT 提取，前端不传，平台管理员跨机构场景除外）。
- PII 静态加密（AES-256），传输加密（TLS 1.3）。
- Phase 1：仅 13 岁以上用户。13 岁以下 COPPA 合规推迟到 Phase 3。

## 0.7 质量门禁与命令

# 下篇 — 产品设计

## 1. 执行摘要

### 1.1 产品定位

**星伴英语是一个 AI 英语私教——会说、会听、会读、会写、会自适应。每个学生都有一个 7×24 小时在线的私人语言教练，费用仅为真人外教的十分之一。**

### 1.2 从传统 SaaS 到 AI 原生

| 维度 | 传统教培 SaaS | 星伴 v4.0（AI 智能体） |
|-----------|------------------------|---------------------------|
| AI 定位 | 辅助工具——写草稿、给建议 | **核心——自主教学引擎** |
| 产品本质 | 机构运营平台 | **AI 英语老师 + 轻量运营支撑** |
| 首要用户 | 创始人 → 老师 → 家长 → 学生 | **学生 → 家长 → 老师 → 创始人** |
| 核心闭环 | 消课 → 扣费 → 续费 | **评测 → 教学 → 练习 → 反馈 → 自适应** |
| 技术重心 | RBAC、对账、计费 | **语音识别、TTS、大模型、自适应算法** |
| 竞争对标 | 校宝、小麦助教 | **多邻国、Speak、Elsa Speak、可汗学院 AI** |

### 1.3 我们造的是什么

六大 AI 引擎由自适应路径规划器统一调度，把每个学生当作独一无二的学习者：

```
┌──────────────────────────────────────────────────────────┐
│                自适应路径规划器                              │
│          （学生能力模型 → 每日任务）                          │
├──────────┬──────────┬──────────┬──────────┬──────────────┤
│ 🎤 口语   │ 📖 阅读   │ ✍️ 写作   │ 🔁 复习   │ 🏆 激励      │
│  引擎     │  引擎     │  引擎     │  引擎     │  引擎        │
├──────────┴──────────┴──────────┴──────────┴──────────────┤
│                  📊 家长报告引擎                            │
└──────────────────────────────────────────────────────────┘
```

## 2. 问题与愿景

### 2.1 痛点与解法

| 痛点 | 现状 | AI 解法 |
|------------|--------------|-------------|
| 不敢开口，怕说错被人笑话 | 学生沉默好几年 | AI 不带评判，无限耐心，无限重试 |
| 一刀切的学习材料 | 能力强的无聊，能力弱的跟不上 | 自适应内容精确匹配当前水平 |
| 没有可执行的反馈 | "你做了 50 道题"——但哪里薄弱？ | 精细诊断："过去式错误率 62%" |
| 真人外教太贵 | ¥200–500/小时，难约，质量参差 | AI：约 ¥0.16/分钟，7×24，质量一致 |
| 没有长期记忆 | 错误反复出现 | FSRS 间隔重复引擎 |
| 坚持不下去 | 自己学 7 天就放弃 | 游戏化循环、连胜、剧情驱动 |

### 2.2 愿景

> **每个学生都有一个 AI 英语私教。**
>
> AI 就是老师。真人老师变成导师——看数据、给情感支持、只在 AI 标记了它解决不了的问题时才介入。
>
> 机构只负责守住质量、留存、续费与风控。

### 2.3 目标学段

| 学段 | 年龄 | CEFR 等级 | 入口体验 |
|---------|-----|------------|-------------|
| 小学（G4–G6） | 9–12 | 入门 → A2 | 游戏化口语 + 自然拼读 |
| 中学（G7–G12） | 12–18 | A1 → B2 | 听说读写全技能 + 备考 |
| 成人 / 考试 | 18+ | B1 → C1 | 雅思/托福口语与写作 |

### 2.4 低能力/高焦虑场景化设计

| 用户画像 | 问题 | 系统策略 |
|-------------|---------|-----------------|
| 低情商 | 不善与老师/家长沟通 | AI 话术模板；语气中立不评判 |
| 学习慢 | 无法把任务拆成步骤 | 错题分解到最小可执行动作 |
| 低执行力 | 无法自我驱动 | 每日最小任务 + 提醒 + 连胜奖励 |
| 急于求成 | 死记硬背不看效果 | 结果以成长证据衡量，不看时长 |
| 内卷/焦虑 | 总跟别人比 | 平台只展示个体进步；无公开排名 |

### 2.5 教育范式对标

| 场景 | 行业参照 | 星伴落地方式 |
|----------|-------------------|------------------------|
| 学生任务闭环 | 多邻国 / ABCmouse 短时任务 | 每日 1–3 个 AI 生成微任务，完成后即时推送下一个 |
| 进度可视化 | 多邻国成长路径 | 里程碑徽章、连胜、弱点雷达图 |
| 复习策略 | ABCmouse 模块化复习 | 错题复活 = 错误标签 + FSRS 间隔重复 |
| 家长沟通 | 家校桥体系 | 可读化周报 + 家长摘要；不展示原始日志 |
| 老师工作流 | 教育 CRM | "异常触发→人工复核→干预处理"闭环 |

## 3. 成功指标

### 3.1 北极星指标

> **学生日均有效学习时长（分钟）**

### 3.2 KPI 目标（第一阶段）

| 指标 | 目标 | 测量方式 |
|--------|--------|-------------|
| DAU/MAU 比 | ≥ 40% | 埋点事件 `session_start` |
| 日均学习时长 | ≥ 15 分钟 | `sum(duration_sec) / active_days / users` |
| 7 日留存率 | ≥ 60% | 同期群：`注册 → 第 7 天活跃` |
| 语音评分 vs 人工评分相关性 | r ≥ 0.85 | 标注测试集，皮尔逊相关系数 |
| 自适应推荐采纳率 | ≥ 70% | `已完成任务 / 已推荐任务` |
| 对话参与度（平均轮次） | ≥ 8 轮 | `对话中用户消息数` |
| 异常处理 SLA（24h） | ≥ 95% | `关闭时间 - 打开时间 ≤ 24h` |
| 审计闭环完整率 | 100% | 关键动作必须在审计日志中有 `trace_id` |
| 无崩溃会话率 | ≥ 99.5% | Firebase / Sentry |
| 家长 NPS | ≥ 50 | 应用内问卷，季度 |

## 4. 用户画像与旅程

### 4.1 用户层级

| 角色 | 优先级 | 核心动机 | 设备 |
|---------|----------|------------|--------|
| **学生**（4–16 岁） | P0 ⭐⭐⭐⭐⭐ | "学好英语，不丢人。" | Web PWA（响应式，可安装） |
| **家长** | P1 ⭐⭐⭐ | "知道孩子有进步，不需要我来教。" | 微信小程序 |
| **老师** | P2 ⭐⭐ | "知道哪些学生需要我，不用批改每份作业。" | Web（桌面端） |
| **管理员（创始人）** | P3 ⭐ | "管理订阅，看全局数据。" | Web（桌面端） |

> **v4.0 核心转变**：学生是第一用户。老师和创始人是"观察者"，不是"操作者"。

### 4.2 学生旅程

```
第 0 天 — 注册体验
  ├─ 15 分钟水平评测 → CEFR 等级 + 能力雷达图
  └─ 分配第一个任务

每日 — AI 今日计划
  ├─ 打开应用 → "早上好！今天为你准备了 3 个任务："
```

## 7. 智能体行为设计

### 7.1 智能体人格原则

每个星伴智能体都遵循一套共同的行为准则：

| 原则 | 说明 |
|-----------|-------------|
| **耐心** | 永远不表现不耐烦；无限重试；庆祝小进步 |
| **精确** | 给具体反馈，不给模糊鼓励（"你的 /θ/ 音需要舌尖放在齿间"而不是"再努力一点"） |
| **私密** | 在学生可见的输出中，永远不对比其他学生 |
| **主动** | 预判学生接下来需要什么；不等学生提问 |
| **保护** | 发现问题时主动标记（麦克风异常、内容太难、参与度下降） |

### 7.2 各智能体人格定义

#### StarVoice 🎤 — 鼓励型教练

- **语气**：温暖、有能量、不带评判
- **行为**：先表扬努力→指出具体错误→示范→邀请重试
- **典型话术**："很接近了！'th' 这个音需要舌尖放在上下齿之间。跟我来：'th-ink'。轮到你了！"
- **绝不说的**："错了。""很差。""这你应该会的。"

#### StarRead 📖 — 好奇型向导

- **语气**：好奇、苏格拉底式、温和挑战
- **行为**：先提问再给答案；连接学生的兴趣点
- **典型话术**："你觉得这个词从上下文来看是什么意思？""猜得不错！它实际上是……"
- **绝不说的**："再读一遍。""这不是很明显吗。"

#### StarWrite ✍️ — 建设型编辑

- **语气**：专业、具体、以改进为导向
- **行为**：先找写得好的地方→指出模式（不是单个错误）→建议升级→展示前后对比
- **典型话术**："你的论点很清楚。让我们让它更有力：这里的 'good' 可以换成 'beneficial'，因为……"
- **绝不说的**："写得不好。"模糊的"需要改进。"

### 7.3 系统提示词模板（Phase 3 — 真实大模型）

#### StarVoice 对话提示词（示例）

```
你是 StarVoice（星语官），一位鼓励型的英语口语教练。
你面对的学生：{name}，{age} 岁，中文母语，当前 CEFR 水平 {level}。
当前场景：{scenario}
已知薄弱音素：{weak_phonemes}

规则：
1. 用学生能听懂的词汇说话。每轮引入 1–2 个新词。
2. 如果学生读错了薄弱音素列表里的音，在你的回复末尾温和地纠正。
3. 每轮保持简短（1–2 句话）。提出一个问题推动对话继续。
4. 每 5 轮对话后，总结：1 个做得好的地方 + 1 个需要练习的地方。
5. 绝不说"错了""不好"，绝不与其他学生比较。
6. 学生卡住超过 5 秒，给提示。
7. 输出格式：5 轮内只输出对话内容；第 5 轮末尾附加 JSON 纠错块。

语言：纯英文（纠错解释可中英混合以助理解）。
```

### 7.4 内容安全规则

| 规则 | 执行 |
|------|-------------|
| 禁止输出当前学生以外任何人的 PII | 预检+事后扫描 |
| 禁止输出涉及自残、暴力、违法的内容 | 禁词列表 + LLM 安全分类器 |
| 禁止输出成人/色情内容 | 安全分类器 |
| 禁止向教学内容注入政治、宗教、争议话题 | 内容生成主题白名单 |
| 所有标记输出 → 人工审核队列 → 4 小时内处理 | 管理员看板队列 |

| 国际音标 | IPA | G1–G6 | 音素听辨、发音部位、最小对立 |
| 自然拼读 | PHONICS | G1–G4 | 字母-发音映射、音节拆分、拼合 |
| 牛津英语 | OXF | G1–G9 | 分级阅读、词汇、语法 |
| 新概念英语 | NCE | G4–G12 | NCE 1–3 册：短文、练习、写作 |
| 课内同步 | K12 | G1–G12 | 单元对齐词汇、句型、备考 |
| 机构自建 | CUSTOM | 全部 | 机构上传内容 |

### 8.2 内容标签强制要求

每个学习任务必须携带以下元数据字段：

```
courseSource   ∈ {IPA, PHONICS, OXF, NCE, K12, CUSTOM}
gradeBand      ∈ {G1-G2, G3-G4, G5-G6, G7-G9, G10-G12}
cefrLevel      ∈ {Pre-A1, A1, A2, B1, B2, C1}
unitId         — 来源定义单元标识
moduleId       — 子单元分组
skillFocus     ∈ {speaking, reading, writing, listening, vocabulary, grammar}
```

### 8.3 内容生成流水线

```
课程源
  → AI 内容适配器（按来源）
  → 分级引擎（对齐学生 CEFR + 蓝思值）
  → 任务生成器（口语 / 阅读 / 写作 / 复习）
  → 质量过滤器（准确性、安全性、适龄性）
  → 任务队列（由 StarPath 分配给学生）
```

### 8.4 Phase 1 内容范围

Phase 1 聚焦精选子集验证流水线：

- 100 个 IPA 发音练习
- 50 篇牛津分级阅读（1–3 级）
- 30 篇新概念 Book 1 课文任务
- K12 同步仅覆盖 G4–G6（人教版 PEP）

完整课程体系在 Phase 2 扩展。

| 家长报告加载（P95） | < 2s |

### 11.2 安全与合规

| 需求 | 实现 |
|-------------|---------------|
| 认证 | Bearer JWT + refresh token；登录失败锁定+限频 |
| 会话策略 | Web 端：Cookie（HTTP-Only）；API 客户端：安全 Token 存储 |
| 授权 | 五级 RBAC |
| 数据隔离 | 所有查询带 `institution_id`（JWT 提取） |
| 字段脱敏 | 老师角色看家长手机：`138****1234` |
| 审计日志 | `actor_id / role / institution_id / action / object_id / trace_id / result / timestamp` |
| PII 加密 | 静态 AES-256；传输 TLS 1.3 |
| COPPA | Phase 1：仅 13+ 用户。13 岁以下合规推迟到 Phase 3 + 律师审核 |
| 异常登录检测 | 审计日志：`actorId / role / institutionId / ip / ua / request_id / trace_id` |

### 11.3 可靠性

| 需求 | 实现 |
|-------------|---------------|
| 离线模式 | 无网络可访问已缓存内容 |
| 弱网络 | 音频本地录制→压缩→排队上传 |
| 优雅降级 | AI 超时 → 模板/缓存兜底 |
| 幂等性 | 所有写接口支持 `Idempotency-Key`；写失败：指数退避，最多 3 次重试 |
| 非 2xx 写操作 | 必须生成审计日志；必须向客户端提示恢复动作 |

### 11.4 平台支持

| 客户端 | 平台 | 优先级 |
|--------|----------|----------|
| 学生端 | Web PWA（Chrome/Safari/Firefox 最新 2 版） | P0 |
| 老师端 | Web（Chrome/Safari/Edge 最新 2 版） | P0 |
| 管理后台 | Web（Chrome/Safari/Edge 最新 2 版） | P1 |

### 12.2 关键架构决策

| 决策 | 选择 | 理由 |
|----------|--------|-----------|
| Serverless vs 容器 | Serverless（Cloudflare） | 零运维开销；可扩至百万用户 |
| 静态托管 | Cloudflare Pages | Git 关联自动部署；全球 CDN |
| 构建工具 | Vite | 极速 HMR；优化生产构建；原生 TS/JSX |
| UI 组件库 | shadcn/ui（Radix + Tailwind） | 无障碍基础组件；copy-paste 所有权；CSS 变量主题化 |
| 样式方案 | Tailwind + shadcn Token | **零自定义 CSS 文件** |
| 实时通信 | WebSocket via Durable Objects | 有状态、低延迟、Cloudflare 原生 |
| AI 适配器 | 统一供应商接口 | 切换大模型供应商适配层外零代码改动 |
| 间隔复习 | FSRS（开源） | SOTA；优于 SM-2 |
| 离线语音 | 本地录制 + 延迟上传 | 核心口语流程不依赖网络 |

### 20.4 设计硬约束（不可谈判）

1. **Figma 冻结后再写代码**——没有审过的设计稿，不写一行代码。
