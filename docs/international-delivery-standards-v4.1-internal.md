# Aggie速记英语（StarMate）v4.1 内部版
## 国际标准交付门禁索引（执行版）

> 本文不是新的 PRD，也不替代设计、研发 SOP 或法务文件。本文只定义 Phase 1 内部版在交付前必须满足的国际化质量、合约、体验、安全与证据门禁。

### 1. 适用范围与业务边界

- 适用阶段：Phase 1 内部可用 Web/H5/PWA 版本。
- 适用对象：本机构内部学生、老师、家长、创始人/运营。
- 公开端边界：只允许课程查看、咨询提交、AI/规则化答疑、试听预约与线索跟进。
- 收费边界：只记录线下/人工收费、课时余额、消课、请假保留与对账；不做在线支付、公开购买、套餐售卖、发票和多机构 SaaS 商用。
- 数据边界：生产路径必须来自 `/api/v1`、D1、R2 或已声明的后端规则结果；前端静态 mock 只能用于 Figma、演示、失败退化或本地 seed，不得作为主要体验来源。

### 3. 国际标准基线

| 维度 | 目标标准 | Phase 1 可验收口径 |
| --- | --- | --- |
| 可用性 | Nielsen 可用性原则 | 用户能在登录、学习、反馈、干预、对账、咨询链路中看见明确下一步和结果状态 |
| 可访问性 | WCAG 2.1 AA 最小口径 | 375px 不横向溢出；文本可读；表单有标签/错误提示；关键操作可键盘触达 |
| 接口质量 | OpenAPI 3.0.3 | 所有业务接口统一 `/api/v1`，请求/响应/错误/鉴权在 `openapi-v4.1.yaml` 可追踪 |
| 数据质量 | D1 单一可信业务源 | 关键学习、课程、课时、线索、收费记录、审计数据可落库、可查询、可导出 |
| 安全合规 | 最小权限、输入验证、审计日志 | 家长只看绑定孩子；老师只看授权学生；公开端不能读内部学生/家长/收费数据 |
| 交付流程 | 测试门禁、审计门禁、运行证据 | 合同校验、构建、烟测、截图/录屏、接口请求证据必须齐全 |

### 4. 控件-接口-字段强制规则

新增、修改或删除任何按钮、输入框、筛选器、弹窗、统计卡、导出入口、AI 操作入口时，必须同时检查并同步：

1. `docs/ui-control-api-field-map.csv` 是否存在控件行。
2. `docs/openapi-v4.1.yaml` 是否存在对应接口、方法、请求字段、响应字段和错误响应。
3. `schema.sql` 是否存在对应 D1 字段、表、索引或审计记录。
4. 前端是否包含 Loading、Empty、Error、Success 四态。
5. 权限是否符合角色边界。

未完成上述同步的控件不得上线。没有真实 API 或明确路由的控件，应删除或降级为不可交互信息展示。

控件-接口-字段映射模板：

| 页面 | 控件 ID | 控件类型 | 用户动作 | API | 方法 | 请求字段 | 响应字段 | D1/R2 字段 | 权限 | 状态要求 | 失败处理 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| `/student/today` | `student.task.submit` | button | 提交练习 | `/api/v1/student/voice-practice/assess` | POST | `taskId,audioFile` | `score,feedback,nextTask` | `learning_events,r2_audio_key` | student | Loading/Error/Success | 保留录音状态并提示重试 |

异常路径模板：

| 链路 | 异常场景 | 必须表现 | 后端要求 | 审计/追踪 |
| --- | --- | --- | --- | --- |
| 登录 | 密码错误、token 过期、角色无权限 | 明确错误原因，可重新登录 | 返回 401/403 与 `request_id` | 记录失败原因和 IP/账号标识 |
| 学生提交 | 网络失败、录音上传失败、评分失败 | 保留任务与录音状态，允许重试 | 不生成重复学习事件；必要时使用 `Idempotency-Key` | 记录任务 ID、失败阶段 |
| 老师干预 | 学生不在授权范围、重复提交 | 禁止提交并显示权限/重复原因 | 权限校验先于写入 | 记录老师 ID、学生 ID、拒绝原因 |
| 家长查看 | 访问非绑定学生 | 不展示任何学生数据 | 返回 403，不返回部分敏感字段 | 记录家长 ID、目标学生 ID |
| 公开咨询 | AI 失败、预约时间无效、隐私未确认 | 降级为人工跟进或表单校验提示 | 写入线索状态，不暴露内部数据 | 记录来源、同意状态、失败阶段 |
| 创始人对账 | 导出失败、字段缺失、权限不足 | 给出失败原因和补救步骤 | 导出前校验字段和角色 | 记录导出人、范围、时间 |

### 5. 角色闭环验收标准

| 角色 | 必须闭环 | 通过标准 |
| --- | --- | --- |
| Student | 今日任务 -> 练习提交 -> 结果回填 -> 复盘回顾 | 任务来自 API；提交后 D1/R2 有记录；学生能看到反馈和下一步 |
| Teacher | 今日课程/异常 -> 学生查看 -> 干预提交 -> 反馈同步 | 老师只能访问授权学生；干预写入 D1；家长/学生端能看到允许范围内的结果 |
| Parent | 孩子概况 -> 课程/课时 -> 成长摘要 -> 沟通反馈 | 家长只能看绑定孩子；缴费/课时为内部记录展示，不出现在线支付入口 |
| Founder/Operations | 看板 -> 风险 -> 线索接管 -> 课时/收费对账 -> 导出 | 数据来自真实接口；接管、转换、对账、导出有审计记录 |
| Public | 课程展示 -> 咨询提交 -> AI/规则化回应 -> 试听预约 | 只写入线索/预约；不能访问内部学生、家长联系方式、收费明细和内部备注 |

### 6. API 与数据合同门禁

- 所有业务接口必须使用 `/api/v1` 前缀。
- 登录链路必须覆盖 `POST /api/v1/auth/login`、token 持久化、`GET /api/v1/me`、会话过期处理和退出登录。
- 成功响应必须可携带 `code`、`message`、`data`、`trace_id`、`request_id`。
- 失败响应必须可携带 `code`、`message`、`error` 或 `detail`，不得只返回空白失败。
- 写操作必须具备输入验证、权限校验和可追踪审计；必要时使用 `Idempotency-Key`。
- D1 为 SQLite 体系，第一阶段不得使用 `JSONB` 假设。
- 涉及课程、课时、收费、学习事件、AI 使用、线索转换、账号状态的字段变更，必须同步 `schema.sql`、OpenAPI 和控件映射表。

### 7. 体验与可访问性门禁

- 首屏 2 秒内必须有可感知内容或明确加载状态。
- 375px、390px、768px、1440px 宽度下不得出现主要内容横向溢出。
- 关键按钮不能被遮挡、误触或仅靠颜色表达状态。
- 表单必须有字段级错误提示，失败后保留用户已输入内容。
- 长文本、金额、日期、状态枚举必须有统一格式，不得出现 `undefined`、`NaN`、空白金额或未翻译状态。
- AI 结果必须标注来源类型或生成状态；失败时给出可执行下一步，不能让用户误以为操作已完成。

### 8. 安全、合规与未成年人保护门禁

- 家长同意、隐私政策、用户协议、AI 内容合规、版权/IP、数据安全文件必须可追踪到合规索引。
- 未成年人语音、学习记录、家长联系方式、收费记录属于敏感业务数据，必须走最小权限。
- 公开端不得展示学生名单、家长电话、实收金额、欠费状态、内部备注、老师内部干预记录。
- AI 不得直接修改账号权限、收费状态、学生归属、内部配额或审计记录。
- 导出功能必须限制角色、记录导出人、导出时间、导出范围和用途。

### 9. 阶段完成前必须运行的门禁

```bash
npm run lint

npm run build
npm run test

```

运行验收门禁：

```bash

```

如某个门禁因环境缺少密钥、端口占用或 Cloudflare 本地状态不可用而无法执行，交付说明必须写清楚：未执行命令、失败原因、影响范围、下一步处理人和替代证据。

### 10. 发布证据包

每次阶段交付或对外演示前，必须附上以下证据：

1. `git` 提交清单，说明文档、接口、数据库、前端和脚本改动。
2. `/api/v1` 关键接口成功与失败请求示例，至少覆盖登录、角色数据、一个写操作、一个权限拒绝。
3. 本地或真实环境截图/录屏，至少覆盖登录、学生闭环、老师/家长/创始人任一闭环、公开咨询链路、异常处理。
4. `docs/ui-control-api-field-map.csv` 缺口检查结果。
5. `docs/openapi-v4.1.yaml` 与实际 Functions 路由差异检查结果。
6. D1 schema/migration 变更说明和回滚说明。

### 11. 阻塞项与回退规则

出现下列情况，判为阶段阻塞，不得进入下一阶段：

- 空按钮、无效控件、点击无反馈。
- 页面以静态 mock 为主，真实 API 只是装饰。
- 页面字段、OpenAPI、D1 schema、控件映射四者不一致。
- 登录成功但 `/api/v1/me` 或角色路由无法持续识别身份。
- 家长、老师、公开端出现越权数据。
- AI 结果没有来源、失败态、人工接管或审计记录。
- 构建通过但浏览器关键链路不可运行。

违规项分类与 SLA：

| 等级 | 判定标准 | 示例 | 处理 SLA | 发布策略 |
| --- | --- | --- | --- | --- |
| 致命 P0 | 权限、数据、登录、支付/收费记录、未成年人隐私、生产不可用风险 | 越权看到学生数据；登录后 `/api/v1/me` 失效；公开端暴露家长电话 | 4 小时内定位并下线入口，24 小时内修复或回滚 | 阻断发布，已发布则立即回退 |
| 严重 P1 | 核心闭环断裂、控件/API/字段不一致、关键异常态缺失 | 学生提交不落库；老师干预无审计；控件映射缺失 | 1 个工作日内修复 | 阻断阶段验收，可保留内部调试 |
| 一般 P2 | 体验、文案、格式、非关键页面状态不一致 | 金额格式不统一；空态文案不清；移动端轻微拥挤 | 3 个工作日内修复 | 不阻断热修外的低风险内部演示，但必须进入缺口清单 |

回退原则：

- 优先回退最近一次控件/API/字段不一致的改动。
- 数据库变更按 migration 和备份策略回退。
- 涉及权限、收费、未成年人数据泄露风险时，先下线入口，再修复。
- 回退后必须重新运行合同校验和至少一条角色闭环烟测。
