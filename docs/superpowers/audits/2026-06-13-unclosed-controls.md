# 2026-06-13 未闭环控件清单（初版）

范围：只列“可见但仍依赖上游数据 / 权限 / 选中态 / 后端响应”的控件。  
结论先放前面：这一轮没有扫到明显的 dead click，主要剩的是 gating 状态和需要产品确认的提示口径。当前可见招生海报已接入统一素材库，文案口径也已统一到“缴费记录”。

## 平台
- `AI 用量筛选 / 导出`：依赖 `institutionId` 和当前结果行数，空状态时应保持禁用。
- `审计筛选 / 导出`：依赖筛选结果与加载状态，空表时导出应禁用。
- `创建机构`：依赖表单完整性和平台权限，当前是完整闭环。
- `异常机构处理`：依赖当前 warnings 列表，属于条件型闭环。

## 创始人
- `课程筛选 / 收费筛选 / 课时筛选`：依赖当前机构和课程数据，当前是条件闭环。
- `课时调整`：依赖学员 ID、课时值、原因，空原因会被禁用。
- `课程创建 / 编辑`：依赖课程抽屉内字段，已闭环。
- `咨询线索转化`：依赖线索选中态、课程 ID 和金额/状态等字段，已闭环。
- `课时对账 / 收费导出`：依赖当前列表数据，空表时导出应禁用。
- `文化墙上传`：依赖 D1 / R2 可用性，上传失败需有回退提示。

## 老师
- `出勤提交`：依赖当前课程和学生选中态，已闭环。
- `授权范围切换`：属于视图开关，不是外部闭环点。
- `生成反馈 / 生成练习`：依赖当前课程和授权学生列表，空列表时应禁用。
- `学生干预提交`：依赖备注内容，当前已闭环。
- `完成记录 / 结课`：依赖当前课程状态，属于条件闭环。

## 家长
- `切换孩子`：依赖当前绑定学员列表，空绑定时应给出占位提示。
- `导出阶段报告`：依赖课时账户和成长数据，已闭环。
- `生成家校反馈`：依赖当前孩子的消息 / 反馈上下文，已闭环。
- `收费疑问入口`：当前是提示文案，不是独立动作控件。

## 学生
- `提交语音评分`：依赖当前任务和识别稿，空任务时应禁用。
- `提交路径完成`：依赖当前路径节点，已闭环。
- `提交练习复盘`：依赖练习答案和分数，已闭环。
- `公开课程选择 / 试听咨询 / 试听预约`：依赖课程选中、咨询已提交和 `leadId`，是串联闭环。

## 首页
- `刷新公开课程`：依赖公开课程接口可用，当前有 loading / 空态保护。
- `提交咨询`：依赖课程选中和家长姓名，已闭环。
- `提交试听预约`：依赖咨询线索 ID，已闭环。
- `发送 AI 回执`：依赖回执内容和线索 ID，已闭环。
- `打开学习成果馆`：会先刷新再跳转，已闭环。
- `招生海报`：已改为从 `mediaLibrary` 的 `admissions` 分区读取，可单独换图，不改布局。

## 学习成果馆
- `查看首页同步 / 查看更多 / 刷新`：依赖文化墙数据源，当前已闭环。
- `上传图片 / 上传视频 / 重试上传`：依赖权限与上传中状态，已闭环。
- `教学视频 / 图片 / 师资 / 反馈` 展示区：属于展示，不是动作闭环点。

## 当前建议
- 这版可以收口，不再继续扩控制入口。
- 后续如果还要微调，只保留两类动作：`刷新` 和 `提交`，其余改成纯展示或状态提示。
