# 统一素材库设计

## 背景

当前项目里有两类和素材有关的展示位：

1. 首页 / 招生页使用固定的招生海报数组 `ADMISSION_POSTERS`
2. 学习成果馆 / 文化墙使用 `cultureWall.photos`、`cultureWall.videos`

这两套资源目前是分离的。实际问题是：

- 招生图会被写死在前端数组和 `public/assets` 路径里
- 图片、视频更新时要改代码，不适合高频迭代
- 招生展示位和文化墙展示位内容同源，但数据结构并没有统一
- 后台上传入口已经存在，但它只覆盖了文化墙一侧

本次设计目标是把“招生海报”和“文化墙素材”收敛到同一套素材库里，做到：

- 素材可随时新增、替换、下线
- 前端页面只按展示位读取，不再依赖固定文件名
- 招生位与文化墙位共享上传、排序、审核和展示规则

## 目标

- 用一套统一素材数据承载招生海报、文化墙图片和文化墙视频
- 首页招生位继续保持海报墙样式，但数据来源改为素材库
- 文化墙继续展示图片和视频，但数据来源也改为素材库
- 后台支持为每条素材指定展示位、类型、标题、说明和排序
- 现有页面在没有新素材时仍有回退内容，不出现空白

## 非目标

- 不做完整 DAM（数字资产管理）系统
- 不做文件夹、标签树、批量裁剪、在线图片编辑
- 不做复杂审批流
- 不做公开上传到游客端
- 不扩展支付、招生表单或课程购买逻辑

## 方案选择

### 方案 A：继续分两套资源，只把固定文件改成可配置链接

优点：

- 改动小
- 页面逻辑基本不动

缺点：

- 招生页和文化墙仍然是两套库
- 后台仍要维护两份资源
- 后续更新仍然容易重复

### 方案 B：统一素材库，按 `placement` 区分展示位

优点：

- 招生页和文化墙共享同一套资源
- 上传、删除、排序和预览可以统一
- 未来增加新展示位时只需加一个 placement

缺点：

- 需要调整现有文化墙返回结构
- 需要把首页招生位从静态数组迁移到动态数据

### 方案 C：新建独立的 `media_assets` 模块，完全重做接口

优点：

- 语义最干净
- 后续扩展空间最大

缺点：

- 迁移量最大
- 现有文化墙接口和文档要大范围改写
- 不适合当前这次收口

## 结论

采用 **方案 B**。

原因很直接：当前项目已经有文化墙上传链路和数据加载链路，直接把它扩成统一素材库，改动最小、收益最大，也最符合“图片、视频随时更新添加”的实际需求。

## 数据模型

统一资源按“素材条目”建模，核心字段如下：

- `id`
- `placement`：`admissions` | `culture-wall`
- `kind`：`photo` | `video`
- `title`
- `description`
- `mediaUrl`
- `coverUrl`
- `status`：`published` | `draft` | `archived`
- `sortOrder`
- `tags`
- `uploader`
- `createdAt`
- `updatedAt`

### 字段规则

- `placement` 决定素材出现在哪个页面或展示区
- `kind=photo` 可用于招生海报墙和文化墙图片位
- `kind=video` 只进入文化墙和素材管理，不进入首页招生海报网格
- `coverUrl` 对视频必填或可回退到自动封面；对图片可直接等于 `mediaUrl`
- `sortOrder` 越小越靠前
- `status=archived` 的素材不参与前台展示

### 兼容当前数据

现有文化墙结构仍可作为兼容层：

- `cultureWall.videos`
- `cultureWall.photos`
- `cultureWall.teachers`
- `cultureWall.feedback`

其中 `videos` 和 `photos` 在新模型里都可映射为统一素材条目。
`teachers` 和 `feedback` 仍保留为文化墙独立内容，不强行并入媒体条目。

首页的 `ADMISSION_POSTERS` 迁移后不再作为主数据源，只保留为无数据时的 bootstrap 回退。

## 页面行为

### 首页招生位

首页招生海报区改为读取统一素材库中：

- `placement=admissions`
- `kind=photo`
- `status=published`

展示规则：

- 保持当前海报墙视觉
- 按 `sortOrder` 排序
- 默认只显示前 4 条或按现有布局可容纳数量
- 点击打开详情抽屉时展示素材标题、说明、标签和可咨询动作

### 学习成果馆 / 文化墙

文化墙改为读取统一素材库中：

- `placement=culture-wall`
- `kind=photo` 和 `kind=video`
- `status=published`

展示规则：

- 图片和视频仍分区展示
- 顶部精选位优先展示最新或手动置顶素材
- 师资信息和家校反馈保持独立模块，不强行塞进素材库

### 后台上传

上传入口变成统一素材录入：

- 先选 `placement`
- 再选 `kind`
- 再补 `title` / `description` / `tags`
- 提交后回写统一素材列表

招生海报区的管理入口默认推荐 `placement=admissions` + `kind=photo`。
文化墙管理入口默认推荐 `placement=culture-wall` + `kind=photo` / `kind=video`。

## API 设计

### 推荐实现

沿用当前文化墙接口族作为第一阶段承载层，在请求里增加 `placement`，让同一套服务返回统一素材列表。

这样做的原因：

- 现有前端已经接了文化墙加载和上传
- 迁移成本最低
- 不会因为接口大改导致首页和文化墙同时失效

### 读取

- `GET /api/v1/admin/culture-wall?placement=admissions`
- `GET /api/v1/admin/culture-wall?placement=culture-wall`

### 上传

- `POST /api/v1/admin/culture-wall`

表单字段新增：

- `placement`
- `kind`
- `title`
- `description`
- `tags`
- `sortOrder`

### 删除 / 更新

后续仍沿用统一素材资源的更新和删除入口，按 `assetId` 操作。

## 前端结构

### 1. 统一数据适配层

增加一个素材适配器，把后端返回的统一条目转换成前端展示所需结构：

- `admissionsAssets`
- `cultureWallPhotos`
- `cultureWallVideos`

适配层负责：

- 过滤 `placement`
- 过滤 `kind`
- 默认排序
- 生成回退图和封面

### 2. 首页招生模块

当前实现已经把首页招生模块迁移到统一素材库，`ADMISSION_POSTERS` 只保留为无数据时的回退。

首页保留：

- 海报卡片
- 标签筛选
- 详情抽屉
- 立即咨询按钮

但所有内容都来自素材库，而不是写死数组。
当前页面里，首页和学生首页的招生位都复用了同一批 `placement=admissions` 素材。

### 3. 文化墙模块

文化墙上传和读取已经接入统一素材库，`CultureWallSection` 和 `CultureWallPage` 继续沿用同一份上传能力。

文化墙页面继续保留：

- 精选内容
- 图片走廊
- 视频走廊
- 师资展示
- 家校反馈

其中图片和视频都从统一素材库取数。

### 4. 上传控件

上传控件不再只区分“图片 / 视频”，还要能指定展示位。

推荐默认值：

- 招生入口默认 `placement=admissions`
- 文化墙入口默认 `placement=culture-wall`

## 数据流

1. 页面加载时请求统一素材列表
2. 适配层按 `placement` 和 `kind` 切分数据
3. 首页招生位渲染招生素材
4. 文化墙渲染文化墙图片和视频
5. 管理员上传新素材后刷新列表
6. 页面重新按 placement 读取并重排

## 回退策略

如果统一素材库没有数据，前端要回退到现有静态种子数据：

- 招生位回退到当前 4 张海报
- 文化墙回退到现有 mock 数据

回退只用于冷启动，不应成为长期主数据源。

## 错误处理

- 上传失败时保留文件选择，允许重试
- 非法 `kind` 或 `placement` 直接在前端拦截
- 招生位如果拿到视频条目，前端不渲染到网格，并提示需要图片素材
- 视频缺少封面时，使用后端封面或本地生成的默认封面
- 列表读取失败时保留当前已渲染内容，不清空页面

## 文案口径

统一使用以下词：

- 素材库
- 招生素材
- 文化墙素材
- 图片素材
- 教学视频
- 展示位

避免继续出现“固定图片”“写死海报”“只能上传一个文件夹”这类口径。

## 测试

需要覆盖以下场景：

1. 首页招生位能从统一素材库读取 4 张以上图片并正常打开详情
2. 文化墙能同时读取图片和视频
3. 上传一张新图片后，首页和文化墙对应展示位能刷新
4. 上传一个视频后，文化墙视频区可见，招生位不误渲染
5. 没有任何真实素材时，页面能回退到现有种子数据
6. 删除或归档素材后，前台列表自动移除

## 迁移步骤

1. 新增统一素材字段和适配层
2. 将现有 4 张招生海报迁移成 `placement=admissions` 的种子数据
3. 将文化墙 photos / videos 迁移成统一素材条目
4. 改造首页招生展示位读取统一素材库
5. 改造文化墙读取统一素材库
6. 保留旧静态资源作为冷启动回退
7. 更新 UI 控件映射文档和未闭环清单

## 验收标准

- 招生海报不再依赖固定数组作为主数据
- 图片和视频可以在后台随时新增和更新
- 首页和文化墙共用同一套素材数据
- 现有页面无空白、无断链、无固定文件名依赖
- 回退资源只在无数据时使用
