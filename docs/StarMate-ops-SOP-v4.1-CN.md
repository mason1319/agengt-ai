## 1. 运营目标

运营目标不是把系统“展示出来”，而是让本机构真实跑通 AI 英语学习闭环。

运营总原则：**一切设计和运营动作围绕解决问题，方便你、我、他。**

运营边界：允许公开展示课程表、开课时间、适合年级、班型、公开价格和剩余名额；允许 AI 客服接待咨询和预约试听；不允许系统自动收费、自动承诺提分、自动保留名额、自动开通正式学生账号。

| 对象 | 运营要方便什么 |
|---|---|
| 创始人 | 看清招生、收费、异常、学习质量 |
| 老师/运营 | 少统计、少重复解释、少手工跟进 |
| 学生 | 每天知道做什么 |
| 家长/客户 | 看懂课程、价格、试听、孩子进步 |

| 目标 | 说明 |
|---|---|
| 学生每天用 | 每天完成 1-3 个任务 |
| 老师少内耗 | 只处理系统标记的异常学生 |
| 家长看得懂 | 每周能看懂孩子进步和待加强 |
| 创始人可控 | 每天能看到活跃、异常、老师处理、AI 用量 |
| 招生可跟进 | AI 客服先接待，运营/创始人跟进高意向线索 |

## 2. 运营角色

| 角色 | 职责 |
|---|---|
| 创始人 | 最终负责人，看整体质量和风险 |
| 运营管理员 | 账号、学生、家长、老师授权、数据巡检 |
| 招生顾问/运营 | 查看 AI 客服线索、预约试听、人工接管、转正式学生 |
| 老师 | 处理异常、提交干预、必要时联系家长 |
| 家长 | 查看孩子成长摘要，必要时申请老师关注 |
| 学生 | 完成今日任务和复习 |
| 技术维护 | 处理系统、部署、数据、接口和权限问题 |

## 3. 内测启动流程

### 3.1 启动前准备

| 项 | 要求 |
|---|---|
| 学生名单 | 姓名、年级、课程、家长联系方式 |
| 老师名单 | 姓名、负责班级/学生 |
| 家长名单 | 与学生绑定关系 |
| 初始课程 | 当前正在教的单元、音标/自然拼读/课内同步内容 |
| 授权文案 | 家长授权和语音采集说明 |
| 内测范围 | 建议 20-50 名学生 |
| 招生咨询文案 | AI 客服开场、隐私提示、试听说明 |

### 3.2 启动步骤

```text
创建机构
  → 创建创始人/运营管理员
  → 导入老师
  → 导入学生
  → 绑定家长
  → 建立老师-学生授权
  → 配置初始任务
  → 通知学生和家长
  → 配置 AI 客服咨询入口
  → 开始第一周内测
```

## 3.3 招生咨询启动步骤

```text
确认咨询入口
  → 配置 AI 客服话术
  → 配置试听时间段
  → 配置隐私勾选
  → 线索进入看板
  → 运营/创始人跟进
  → 试听完成后转正式学生
```

## 4. 账号运营

### 4.1 账号创建

| 账号 | 创建人 | 规则 |
|---|---|---|
| 创始人 | 技术维护/系统管理员 | 仅 1-2 个 |
| 运营管理员 | 创始人 | 可辅助导入数据 |
| 老师 | 创始人/运营管理员 | 必须绑定授权学生 |
| 学生 | 创始人/运营管理员 | 必须绑定家长 |
| 家长 | 创始人/运营管理员 | 必须绑定学生 |

### 4.2 账号停用

| 场景 | 操作 |
|---|---|
| 学生退课 | 停用学生账号，保留历史数据 |
| 老师离职 | 停用老师账号，转移授权学生 |
| 家长手机号变更 | 更新家长账号并保留绑定关系 |
| 异常登录 | 临时停用账号并通知创始人 |

### 4.3 权限检查

每周检查一次：

| 项 | 标准 |
|---|---|
| 老师授权 | 老师只能看到负责学生 |
| 家长绑定 | 家长只能看到自己孩子 |
| 停用账号 | 停用后不能登录 |
| 家长手机号 | 老师端必须脱敏 |

## 5. 学生运营

### 5.1 每日学生任务

第一阶段学生每日任务控制在 1-3 个，避免任务太多导致放弃。

| 任务类型 | 建议 |
|---|---|
| 口语练习 | 每天 1 个 |
| 复习任务 | 每天 1 个 |
| 跟读/音标 | 按课程安排 |

### 5.2 学生提醒

| 场景 | 处理 |
|---|---|
| 当天未开始 | 系统标记，家长端可见 |
| 连续 3 天未完成 | 进入老师异常看板 |
| 连续低分 | 进入老师异常看板 |
| 重复错同一知识点 | 进入复习队列 |

### 5.3 学生激励

| 机制 | 说明 |
|---|---|
| 连续学习 | 鼓励坚持 |
| 任务完成反馈 | 做完马上看到结果 |
| 错误重试 | 让学生看到进步 |
| 个人成长 | 只和自己比，不公开排名 |

禁止：

| 禁止项 | 原因 |
|---|---|
| 公开排名 | 制造未成年人焦虑 |
| 和其他孩子比较 | 引发家长内卷 |
| 过度弹窗刺激 | 影响学习 |

## 6. 老师运营

### 6.1 老师每日工作流

```text
登录老师端
  → 查看今日异常
  → 进入学生详情
  → 判断是否需要干预
  → 提交干预记录
  → 必要时联系家长
```

### 6.2 老师处理标准

| 异常 | 处理建议 |
|---|---|
| 连续 3 天未完成 | 先给家长提醒，再观察 |
| 口语连续低分 | 安排一次针对性练习 |
| 复习反复错误 | 布置强化任务 |
| 家长申请关注 | 24 小时内回复 |
| 系统误判 | 标记误判并提交原因 |

### 6.3 SLA

| 事项 | 时限 |
|---|---|
| 高风险学生 | 24 小时内处理 |
| 家长申请关注 | 24 小时内回复 |
| 普通低活跃 | 48 小时内处理 |
| 误判标记 | 当天处理 |

## 7. 家长运营

### 7.1 家长首次使用

家长首次登录需要完成：

| 步骤 | 内容 |
|---|---|
| 确认孩子 | 查看绑定学生 |
| 授权确认 | 确认学习数据和语音采集说明 |
| 查看成长卡 | 理解本周学习情况 |

### 7.2 家长沟通原则

| 原则 | 说明 |
|---|---|
| 少术语 | 不展示复杂技术指标 |
| 说结果 | 告诉家长孩子哪里进步 |
| 说下一步 | 告诉家长接下来怎么做 |
| 不制造焦虑 | 不和其他学生比较 |

### 7.3 家长反馈处理

| 反馈 | 处理 |
|---|---|
| 看不懂报告 | 老师/运营补充解释 |
| 申请老师关注 | 进入老师异常看板 |
| 质疑评分 | 老师复核 |
| 要求删除数据 | 创始人审批后技术处理 |

## 8. 创始人运营

### 8.1 每日看板

创始人每天看四项：

| 指标 | 说明 |
|---|---|
| 活跃学生 | 今日有学习行为的人数 |
| 任务完成率 | 今日任务完成情况 |
| 未处理异常 | 老师尚未处理的问题 |
| AI 用量 | 请求数、语音分钟、成本估算 |

### 8.2 每周复盘

| 复盘项 | 目标 |
|---|---|
| 学生留存 | 哪些学生连续低活跃 |
| 教学问题 | 高频错误点是什么 |
| 老师处理 | 谁的异常未处理 |
| 家长反馈 | 哪些家长不理解报告 |
| 成本 | AI 和存储是否超预算 |

### 8.3 决策动作

| 情况 | 决策 |
|---|---|
| 学生活跃低 | 调整任务难度或提醒机制 |
| 口语低分集中 | 老师安排专项练习 |
| 家长反馈差 | 优化周报文案 |
| AI 成本高 | 限制真实模型调用或启用规则化降级 |

## 9. 招生咨询运营

### 9.1 AI 客服工作流

```text
家长/个人进入咨询页
  → AI 客服问候
  → 收集孩子年龄、年级、英语问题、联系方式
  → AI 客服给初步学习方向
  → 推荐试听时间
  → 生成线索
  → 运营/创始人跟进
```

### 9.2 AI 客服话术边界

| 允许 | 禁止 |
|---|---|
| 介绍课程方向和试听流程 | 承诺提分、承诺效果 |
| 收集基础学习情况 | 收集身份证、家庭住址等无关敏感信息 |
| 推荐试听时间 | 自动收费 |
| 说明老师会跟进 | 直接开正式学生账号 |
| 给初步建议 | 替代老师最终诊断 |

### 9.3 线索状态运营

| 状态 | 运营动作 |
|---|---|
| `new` | 24 小时内查看 |
| `contacted` | 记录沟通结果 |
| `trial_scheduled` | 确认试听时间和老师 |
| `trial_completed` | 记录试听结果 |
| `enrolled` | 转正式学生和家长账号 |
| `invalid` | 标记原因 |
| `archived` | 归档 |

### 9.4 转正式学生

转正式学生必须完成：

| 项 | 要求 |
|---|---|
| 家长确认 | 确认孩子信息和联系方式 |
| 家长授权 | 确认未成年人使用和语音采集 |
| 老师分配 | 建立老师-学生授权 |
| 初始任务 | 配置第一批学习任务 |
| 账号通知 | 发送学生/家长登录方式 |

### 9.5 课程表与收费运营

参考课程表图片，运营上分成公开课程展示和内部收费记录两部分。

| 场景 | 内容 | 负责人 |
|---|---|---|
| 公开课程展示 | 课程名、班型、时间段、适合年级、课次、价格、连报价、剩余名额 | 运营 |
| AI 客服推荐 | 根据孩子年级和问题推荐课程 | AI 客服 + 运营复核 |
| 试听预约 | 选择课程和试听时间 | 家长/客户 |
| 内部报名 | 将线索转为报名 | 运营 |
| 收费记录 | 记录线下已收、未收、金额、时间、备注 | 创始人/运营 |

收费展示规则：

| 规则 | 说明 |
|---|---|
| 课程价格公开 | 客户和家长可以看到课程单价、连报价 |
| 个人缴费不公开 | 不展示其他学生缴费状态 |
| 家长只看自己孩子 | 家长可看自己孩子课程和缴费状态 |
| 创始人看全局 | 创始人可看全部课程收费和未收费 |
| 不接在线支付 | 第一阶段只做收费记录 |

课程发布必填项：

| 字段 | 示例 |
|---|---|
| 课程名称 | 自然拼读速记班 |
| 班型 | 小班课 / 大班课 / 一对一 |
| 适合年级 | 一二年级、小升初 |
| 上课日期 | 2026-08-07 至 2026-08-27 |
| 上课时间 | 上午 8:30-10:30 |
| 课次 | 16 次 |
| 单期价格 | 2080 |
| 连报价格 | 3744 |
| 限招人数 | 12 人 |
| 剩余名额 | 自动计算或人工维护 |

### 9.6 每日点名与课时运营

核心目标：家长缴费、学生课时、每天打卡、没来的保留课时都能一目了然。

| 场景 | 运营规则 |
|---|---|
| 到课 | 老师标记到课，系统消耗课时 |
| 迟到 | 老师标记迟到，默认消课，可备注 |
| 请假 | 老师标记请假，不消课，课时保留 |
| 机构停课 | 不消课，课时保留 |
| 无故缺课 | 按机构规则处理，必须备注 |
| 补课 | 完成补课后消耗保留课时 |

老师上课前：

| 动作 | 说明 |
|---|---|
| 打开点名表 | 查看本节课学生 |
| 标记状态 | 到课/请假/缺课/迟到 |
| 提交记录 | 生成消课或保留记录 |

家长可见：

| 信息 | 说明 |
|---|---|
| 已缴金额 | 仅自己孩子 |
| 总课时 | 购买课时 |
| 已上课时 | 已消耗 |
| 剩余课时 | 可继续上课 |
| 保留课时 | 请假/停课未扣 |
| 每次记录 | 日期、课程、老师、扣课/保留原因 |

每周对账：

| 项 | 负责人 |
|---|---|
| 核对收费记录 | 创始人/运营 |
| 核对消课记录 | 老师/运营 |
| 核对剩余课时 | 运营 |
| 异常调整 | 创始人审批 |

## 13. 故障处理

### 13.1 故障等级

| 等级 | 示例 | 处理时限 |
|---|---|---|
| P0 | 无法登录、数据越权、录音泄露 | 立即处理 |
| P1 | 学生无法提交任务、老师无法处理异常 | 当天处理 |
| P2 | 家长报告延迟、AI 反馈失败 | 48 小时内 |
| P3 | 文案、样式、小体验问题 | 排期处理 |

### 13.2 应急流程

```text
发现问题
  → 记录截图/账号/时间
  → 判断等级
  → 通知技术维护
  → 临时降级或停用相关功能
  → 修复
  → 复盘
```

### 13.3 重大风险动作

| 风险 | 动作 |
|---|---|
| 权限越权 | 立即停用相关账号，锁定接口 |
| R2 录音泄露 | 立即关闭公开访问，轮换签名策略 |
| 家长数据错误 | 暂停对应页面，修复绑定关系 |
| AI 输出不适合儿童 | 下线内容，进入人工审核 |
