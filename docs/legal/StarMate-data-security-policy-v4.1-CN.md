# 星伴英语 StarMate 数据安全与留存制度 v4.1（草案）

**适用范围**：D1 数据、R2 文件、AI 用量、审计日志、账号权限  
**状态**：内部草案，正式使用前需技术和法务复核  
**日期**：2026-06-06  

---

## 1. 数据分类

| 等级 | 数据 | 保护要求 |
|---|---|---|
| 高 | 未成年人信息、语音录音、家长手机号 | 严格权限、脱敏、私有存储 |
| 中 | 学习任务、得分、错题、周报 | 角色隔离、审计 |
| 中 | 老师干预记录 | 授权可见、审计 |
| 低 | 系统配置、非敏感统计 | 权限控制 |

---

## 2. 权限规则

| 角色 | 可见范围 |
|---|---|
| 学生 | 自己任务和结果 |
| 家长 | 自己孩子 |
| 老师 | 授权学生 |
| 创始人 | 本机构全部数据 |
| 技术维护 | 必要维护范围，操作受审计 |

---

## 3. 留存规则

| 数据 | 建议留存 |
|---|---|
| 学生档案 | 在读期间 |
| 学习事件 | 在读期间及必要归档 |
| 语音录音 | 默认 30 天滚动 |
| R2 素材 | 按用途设置生命周期 |
| AI 用量日志 | 不少于 6 个月 |
| 审计日志 | 不少于 6 个月 |
| 停用账号 | 保留必要历史后归档或删除 |

---

## 4. 删除流程

```text
家长/用户申请
  → 运营登记
  → 创始人审批
  → 技术执行删除/匿名化
  → 记录审计
  → 反馈处理结果
```

删除前需确认是否存在教学纠纷、账务、审计或法律保留需求。

---

## 5. R2 文件安全

1. R2 默认私有桶。
2. 禁止裸公开学生录音。
3. 访问使用签名 URL 或后端代理。
4. 文件上传需限制类型和大小。
5. 每个文件必须写入 `media_assets` 索引。
6. 录音默认 30 天生命周期。

---

## 6. 审计日志

必须记录：

| 操作 | 字段 |
|---|---|
| 登录 | user_id、role、ip、ua、result、trace_id |
| 上传录音 | student_id、file_key、result、trace_id |
| AI 评分 | provider、latency、result、trace_id |
| 老师干预 | teacher_id、student_id、action、trace_id |
| 家长查看 | guardian_id、student_id、trace_id |
| 数据导出/删除 | operator_id、scope、result、trace_id |

---

## 7. 数据备份

| 数据 | 策略 |
|---|---|
| D1 | 定期导出备份 |
| R2 | 保留生命周期策略，关键素材备份 |
| 配置 | wrangler/env 配置单独保存 |
| 密钥 | 不写入代码仓库 |

---

## 8. 事故响应

| 等级 | 示例 | 动作 |
|---|---|---|
| P0 | 数据越权、录音泄露 | 立即停用相关入口，通知创始人 |
| P1 | 大量登录失败、AI 输出异常 | 限流/降级，排查日志 |
| P2 | 单个用户数据错误 | 修复绑定关系 |

事故处理后必须复盘并更新 SOP。
